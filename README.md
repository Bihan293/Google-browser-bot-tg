# tg-browser-bot

Telegram-бот на **Go**, который работает как мини-браузер / поисковик
(в духе Google) **внутри Telegram**. Бот никуда не «перебрасывает» в
реальный Google — все результаты (сайты, картинки, видео, новости,
содержимое страниц) показываются прямо в чате.

## Возможности

- 🌐 **Веб-поиск** — текстовые результаты с заголовком, сниппетом и URL
  (источник — DuckDuckGo HTML, без API-ключа).
- 🖼 **Картинки** — выдача альбомом (media group) с подписями.
- 🎥 **Видео** — поиск по YouTube (парсинг `ytInitialData`), карточка
  ролика с превью, автором и длительностью.
- 📰 **Новости** — Google News RSS (без API-ключа), последние материалы
  по теме.
- 📄 **Чтение страниц в чате** — пришли ссылку или нажми на номер
  результата → бот достанет читаемый текст и (если есть) обложку.
- ⬅️➡️ Пагинация, переключение режимов кнопками.
- Webhook-режим (Telegram пушит обновления), HTTPS endpoint, health-check.

## Архитектура

```
cmd/bot/                 — точка входа (main)
internal/config/         — загрузка конфигурации из env
internal/bot/            — Telegram-клиент, webhook-сервер, маршрутизация
internal/handlers/       — обработчики сообщений, команд, callback'ов
internal/search/         — провайдеры поиска: web / images / videos / news
internal/render/         — извлечение читаемого контента со страницы
internal/keyboard/       — inline и reply клавиатуры
internal/store/          — in-memory кэш сессий пользователей
```

> База данных **не нужна** — состояние пользователя (последний запрос,
> страница пагинации, кэш результатов) хранится в памяти с TTL 30 минут.
> Этого хватает для типичного UX «искать → листать → открывать».
> При желании `internal/store` легко заменить на Redis/Postgres-реализацию
> того же интерфейса.

## Конфигурация (переменные окружения)

| Переменная        | Описание                                              | Обязательная |
|-------------------|-------------------------------------------------------|--------------|
| `BOT_TOKEN`       | Токен от @BotFather                                   | ✅           |
| `WEBHOOK_URL`     | Публичный HTTPS URL сервера, например `https://x.io`  | ✅ (для прод)|
| `WEBHOOK_PATH`    | Путь webhook'а, по умолчанию `/tg/webhook`            | ❌           |
| `WEBHOOK_SECRET`  | Секрет для проверки заголовка от Telegram             | ❌           |
| `PORT`            | Порт HTTP-сервера, по умолчанию `8080`                | ❌           |
| `DEBUG`           | `true` — подробные логи Telegram API                  | ❌           |

См. `.env.example`.

## Запуск локально

```bash
export BOT_TOKEN=123456:ABC...
export WEBHOOK_URL=https://your.ngrok.io
go run ./cmd/bot
```

## Сборка

```bash
go build -o bin/bot ./cmd/bot
./bin/bot
```

## Docker

```bash
docker build -t tg-browser-bot .
docker run --rm -p 8080:8080 \
  -e BOT_TOKEN=... \
  -e WEBHOOK_URL=https://your.domain \
  tg-browser-bot
```

## Команды бота

- `/start` — приветствие, главное меню
- `/help` — справка
- `/search <запрос>` (`/s`, `/web`) — поиск по сайтам
- `/img <запрос>` — поиск картинок
- `/vid <запрос>` — поиск видео
- `/news <тема>` — поиск новостей
- `/open <url>` — открыть страницу внутри бота

Также можно просто отправлять текст (бот возьмёт последний выбранный
режим) или ссылку — она будет открыта внутри чата.
